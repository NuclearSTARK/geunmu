v5.4.7 - 홈화면 아이콘 흰 배경, 진짜 원인 (아이콘 파일명 변경)

지난 번 스크린샷을 보니 앱 안의 아이콘(왼쪽 위)은 이미 남색으로 잘 나오는데,
"홈 화면에 추가" 미리보기만 계속 흰 배경으로 나왔어요.
이건 아이폰 사파리가 apple-touch-icon을 파일 내용이 아니라 "파일 경로" 기준으로
따로, 아주 오래 캐싱해버리는 문제라서 ?v=버전 숫자를 붙이는 정도로는 안 뚫렸던 거예요.

이번엔 무엇을 바꿨나요
- 아이콘 파일을 아예 새 이름으로 만들었습니다: icon-180-b.png / icon-192-b.png / icon-512-b.png
- index.html, manifest.json이 전부 이 새 파일명을 보도록 수정했습니다.
- 파일명 자체가 새것이라 사파리가 예전에 캐싱해둔 것과 절대 겹치지 않습니다.

적용 순서 (중요)
1. 이 zip 전체를 저장소 루트에 덮어쓰기 업로드 (예전 icon-180.png 등도 그대로 두셔도 되고,
   지우셔도 무방합니다 - 이제 아무데서도 참조하지 않습니다)
2. 홈화면의 기존 "Seul Police" 아이콘 삭제
3. Safari 앱 완전히 종료 후 다시 열기 (최근 앱 목록에서 위로 스와이프)
4. 사이트 재접속 → 공유 → "홈 화면에 추가"

이번에도 흰 배경으로 보이면, 사파리 설정 > 고급 > 웹사이트 데이터에서
nuclearstark.github.io 데이터를 삭제하고 다시 시도해주세요.
